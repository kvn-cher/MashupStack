The Importance of Your Resume


Every person is different.
The HR or the interviewer knows nothing about you.

Even before the interviewer sees you, he sees your resume.
A resume helps to convey who you are to the HR or the interviewer.

Based on your resume, the interviewer forms a perception about you in his mind.
That perception might be right.
That perception might be wrong.
But the point is, that he has already formed a mental image about you in his mind.

Now whether he has thought good about you or bad about you depends on how he interpreted your resume.
And if you can create a good resume, that can help you big time in your interview.

And they rightly say 'Well begun is half done'.
I hope that clarifies the importance of your resume.

So, the resume you create should be simple, neat, readable ( easily understandable ) and free of any spelling/grammar mistakes.

In this topic we will discuss how to create good resume.

Interpreting Your Resume


When you write your resume, you think from an EMPLOYEE PERSPECTIVE.
However when the interviewer reads your resume, he looks at it from a whole new angle.
He sees it from an EMPLOYER PERSPECTIVE.

So your mistakes, unwanted sentences, grammatical errors and untrue statements can cast a BIG SHADOW on your and can infact work against you in the interview.

For example,
- When you have trouble writing decent english, they think if you are good in communicating with a client.
- When you are unable to express your ideas clearly, they think how would you communicate within a team.
- When you lie about a project or experience or skill in your resume, they take you granted for a liar and consider not worthy of hiring into the team. Skill is only so much. Rest is his morale, values, character and attitude.
- When you have spelling mistakes, grammar mistakes and layout issues in your resume, they wonder whether you will be this lazy and less concerned to not even proofread and double check when you deliver a project to the client.

Remember, anything and everything you write in your resume speaks volumes about you
THEY JUDGE YOU BASED ON YOUR RESUME.

Where to Create Your Resume?


There are different ways to create resume. You can create it with
1. MS Word within your computer
2. Any online resume websites

Problems when you create resume in MS Word
If you create a resume in MS Word, in some days you will get confused as to which location it was saved. Sometimes, you get confused with multiple versions of your resume that you will mistakenly send a wrong resume for a job interview. More than that, if you don't have access to your computer ( eg: if you are out of your house, traveling or in a vacation. ), you cannot send your resume to companies.

Problems when you create resume in online resume websites
If you create it in online websites, you will face difficulty while editing it. Sometimes, it has been seen that, at times when we need to download resume to send them, the website will not be available or it might ask for payment ( you might be in a free mode and you wont know it. )

So how should you create a resume?
I recommend another method.
You should follow that method while you are studying in Mashupstack.
You should create resumes with Google Docs.

If you create resume with Google docs,

- You can access it from web or mobile
- It will be available for you everywhere - even if you are outside or on a vacation.

Things to Note While Writing Resume


Try to remember these when you make your resume.
The content provided in the sample resume are only examples. Mere samples.
You SHOULD NOT keep the same sentence.
You SHOULD CHANGE those sentences.
We don't want everyone to have the same resume right?
Just try to keep the format roughly the same.


One Line Summary


Some of the examples include:-
FullStack Web Developer - Python - ( Fresher )
FrontEnd Developer - Angular (Fresher )
Backend Developer - PHP (Fresher )

If you are studying both frontend and backend, then it's good to write 'Fullstack Web Developer - (Technology) - Fresher'
If you are making the resume after learning ONLY HTML, CSS, Bootstrap, write 'Web Designer - Fresher' instead.
If you are making the resume after learning ONLY HTML, CSS, Bootstrap, Javascript and jQuery, write 'Frontend developer - Fresher' instead.
If you have learnt front end with angular, mention 'Frontend developer - Angular - Fresher'

As most of the resumes that they get are NOT Fullstack developer resumes, this word should catch their eye.
If you only write 'Fullstack Web Developer - ( PHP )', they might confuse you as an experienced person. So please include the fresher word also.

If you only write, 'Fullstack Web Developer - Fresher', some HR might not understand it. They might be looking for a PHP developer and they might miss your resume. So its always better to include the main technology. So write it as 'Fullstack Web Developer - ( PHP ) Fresher'.

GitHub URL


Include the Github url only if you have one. Ideally you should be having one.

I haven't really seen people keep this. And this is NOT expected usually from a fresher.
And that is EXACTLY WHY you need to keep it. ( if you have one )

Hopefully, the HR or the Technical lead who checks your resume knows that Github is for uploading code.
And since you have a github profile, they get a feeling that you are into coding. That's a BIG advantage.

To the interviewer, this gives the following perception ( and maybe more.. )
- you are interested in coding
- you are up to date with latest trends like version control with GIT
- you have already worked in teams or in such projects
- its easy to induct you into the team as you don't hopefully need elaborate training
- And since you have already worked in team, it becomes easy for you to get along

However, Include this ONLY IF
- you know GIT
- you have code in GIT

Ideally, all your exercises and project that you do as part of your MashupStack learning, SHOULD BE IN GITHUB.

You can ignore this if
- you don't have a github url
- you are not comfortable keeping it
- you just don't want it to be included.

And that's all perfectly okay.

Skills


I prefer this section to come before your education.
This skills section should have a LOT of skills and should show up in the first page of the resume
As soon as the interviewer sees your resume, he should see your skills instantly even without having to scroll down the page.

Include whatever technical skills you have.
If you don't have the skills mentioned in that resume, remove them from the resume.
Remember: Interviewer can ask from any area that you have kept in the resume. So if you don't know a technology it is best if you just remove it altogether from the skill list.

Education


Always provide the recently completed education qualification first and then followed by older ones.
So your Graduation details will come first, then your Plus Two academic details and then the 10th academic details.

Project


Include your project in college.
If you have done any project at Mashup ( good ones ONLY ) then include that also.
You can remove this area if you don't want it there.

Mini Project


Include your college mini project. You can remove this area if you don't want it there.

Seminar


Include your seminar topic. You can remove this area if you don't want it there.

Achievements


You should definitely have some achievements.
If you don't have any, create some dummy stuff which doesn't need certificates to be presented.

Personal Qualities


Add some qualities that you think you have.

Personal Details


Include all your necessary personal details.
Don't include your passport number.
Don't include your marital status or even your blood group.

Contact Details


Include your contact details.
If you present and permanent address are the same, don't repeat it twice. Just mention Address.

This section is not really needed. But you can keep it if you want.
